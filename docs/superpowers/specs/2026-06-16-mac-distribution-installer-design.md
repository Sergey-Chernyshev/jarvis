# Дизайн: macOS-дистрибутив Jarvis (подписанный DMG + updater + онбординг)

Дата: 2026-06-16
Статус: утверждён к реализации

## Цель

Превратить Jarvis из «склонируй и `npm run`» в полированное Mac-приложение,
которое можно выложить на сайт и в GitHub: пользователь скачивает подписанный
и нотаризованный `.dmg`, перетаскивает `Jarvis.app` в `/Applications`, при первом
запуске приложение само предлагает доустановить интеграцию с Claude Code и
голос Silero, а дальше обновляется через встроенный updater.

## Не-цели

- Лендинг/сайт загрузки — пользователь делает сам, мы отдаём только артефакты.
- Mac App Store (там песочница несовместима с установкой хуков в `~/.claude` и
  спавном `tmux`/`claude`/`python`). Распространение — только Developer ID вне Store.
- Windows/Linux сборки — отдельная тема, сейчас только macOS.
- Бандлинг PyTorch внутрь DMG — он остаётся runtime-загрузкой в venv (сотни МБ–ГБ),
  тащить его в образ нельзя и не нужно.

## Ключевые решения

- **Подпись/нотаризация**: Developer ID Application + hardened runtime +
  `notarytool` + степлинг. У пользователя есть платный Apple Developer ID.
- **Архитектура бинаря**: universal (`x86_64` + `aarch64`) — чтобы DMG ставился
  на любой Mac, скачанный с сайта.
- **Обновления**: `tauri-plugin-updater`. Манифест `latest.json` в GitHub Release,
  подпись артефактов updater-ключом. Приложение проверяет обновления на старте.
- **Сборка**: GitHub Actions на тег `v*` через `tauri-apps/tauri-action`. Секреты
  (сертификат, пароли, Apple ID/team, updater-ключ) — в GitHub Secrets.
- **Онбординг «умно»**: логика установки интеграции выносится в общий lib-модуль,
  который вызывают и CLI-бинарь `jarvis-setup`, и приложение. Первый запуск без
  установленной интеграции → окно с кнопкой и живым логом прогресса. Установка
  идёт **в процессе приложения** (не через sidecar-бинарь — меньше возни с путями),
  прогресс стримится в окно событиями Tauri.

## Архитектура

Две фазы. Фаза 1 даёт работающий скачиваемый DMG; Фаза 2 добавляет «умный»
первый запуск. Фазы независимы по риску, но входят в один релиз.

### Фаза 1 — дистрибутив

1. **Бандл-конфиг** (`src-tauri/tauri.conf.json`)
   - `bundle.active: true`, targets `dmg` + `app`.
   - `bundle.icon` — полный набор из `tauri icon` (исходник 1024×1024 PNG).
   - `bundle.macOS`: `minimumSystemVersion`, hardened runtime, entitlements-файл,
     `signingIdentity` (из env в CI), кастомный фон DMG с drag-стрелкой в `/Applications`.
   - `bundle.createUpdaterArtifacts: true` — чтобы собрался `.app.tar.gz` + подпись.
   - `plugins.updater`: `endpoints` (GitHub Release latest.json), `pubkey`.

2. **Иконка**
   - Вход: `src-tauri/icons/icon-source.png` (1024×1024). Если пользователь не даёт
     свой — генерируем аккуратный плейсхолдер-логотип, заменяемый позже.
   - `tauri icon` раскладывает `.icns` + PNG-набор в `src-tauri/icons/`.

3. **Entitlements** (`src-tauri/entitlements.plist`)
   - Минимум: сетевой клиент (updater). Спавн дочерних процессов
     (`python`/`tmux`/`claude`) под hardened runtime разрешён без спец-энтайтлментов;
     PyTorch грузится в отдельном venv-python, под энтайтлменты приложения не попадает.

4. **Updater-ключи**
   - `tauri signer generate` → приватный ключ в GitHub Secret
     `TAURI_SIGNING_PRIVATE_KEY` (+ пароль), публичный — в `plugins.updater.pubkey`.

5. **CI** (`.github/workflows/release.yml`)
   - Триггер: `push` тега `v*`.
   - Раннер `macos-14`, Rust + Node, target `universal-apple-darwin`.
   - Импорт Developer ID сертификата из секретов во временный keychain.
   - `tauri-apps/tauri-action`: сборка → подпись → нотаризация → степлинг →
     генерация updater-манифеста → создание GitHub Release (черновик) с
     `Jarvis_x.y.z_universal.dmg`, `Jarvis.app.tar.gz`, `Jarvis.app.tar.gz.sig`,
     `latest.json`.
   - Секреты: `APPLE_CERTIFICATE`, `APPLE_CERTIFICATE_PASSWORD`,
     `APPLE_SIGNING_IDENTITY`, `APPLE_ID`, `APPLE_PASSWORD` (app-specific),
     `APPLE_TEAM_ID`, `TAURI_SIGNING_PRIVATE_KEY`, `TAURI_SIGNING_PRIVATE_KEY_PASSWORD`.
   - README: раздел «Релиз» с пошаговым заведением секретов и `git tag`.

### Фаза 2 — онбординг при первом запуске

1. **Вынос логики установки в lib** (`src-tauri/src/install/mod.rs`)
   - Переносим `install` / `uninstall` / `status` и хелперы из `bin/setup.rs` в
     модуль `install`, callable и из бина, и из приложения.
   - `bin/setup.rs` ужимается до тонкой CLI-обёртки над `install::*`.
   - Установочные шаги принимают колбэк прогресса `Fn(&str)` — для стрима в UI.
   - `install::status()` возвращает структуру (что стоит/не стоит), а не печать.

2. **Окно онбординга**
   - При старте приложение зовёт `install::status()`. Если интеграция не
     установлена (нет хуков/шима) → открывает окно `onboarding`.
   - Окно: заголовок, одна кнопка «Настроить Jarvis», область живого лога.
     Кнопка запускает `install::install(progress_cb)` в отдельном потоке; каждая
     строка прогресса (хуки → шим → tmux → Silero/PyTorch) эмитится событием
     Tauri в окно. По завершении — «Готово», окно можно закрыть.
   - Silero-шаг тяжёлый (pip/torch): показываем явный «ставлю PyTorch, это надолго».
     Любая не-фатальная ошибка (Piper уже выпилен; Silero) — лог + приложение
     работает дальше, как и в CLI.

3. **Пункт меню «Переустановить интеграцию»**
   - В tray-меню: повторно открывает онбординг (ремонт/переустановка).

## Потоки данных

- **Релиз**: `git tag v0.2.1 && git push --tags` → CI собирает/подписывает/
  нотаризует → черновик GitHub Release с артефактами → ты публикуешь.
- **Обновление**: приложение на старте читает `latest.json` из последнего
  Release → если версия выше → предлагает скачать/поставить (подпись проверяется
  публичным ключом).
- **Онбординг**: первый запуск → `status` пуст → окно → кнопка →
  `install(progress_cb)` → события прогресса → готово.

## Обработка ошибок

- Сборка/подпись падает в CI → релиз не создаётся, лог в Actions.
- Нотаризация отклонена → шаг падает явно (стандарт Apple), фикс по логу notarytool.
- Updater недоступен/нет сети → приложение работает как обычно, ошибка в лог.
- Онбординг: каждый шаг fail-safe; Silero/сеть могут отвалиться — интеграция
  (хуки) всё равно ставится, голос просто молчит. Демон не падает.

## Тестирование

- `cargo test` — существующие тесты + тесты `install::status()` (детект
  установлено/нет на временном `$HOME`).
- Ручной прогон: локальная подпись+нотаризация скриптом (см. ниже) на чистом
  пользователе/VM → DMG открывается двойным кликом без предупреждений.
- Updater: собрать `v0.2.0` и `v0.2.1`, проверить, что 0.2.0 видит и ставит 0.2.1.
- Онбординг: запустить .app на машине без `~/.jarvis` → окно → установка → хуки в
  `~/.claude/settings.json`, шим в PATH, Silero поднимается.

## Необходимые входы от пользователя

- Исходная иконка 1024×1024 (иначе — плейсхолдер).
- Apple Developer ID сертификат (экспорт `.p12`) + пароль, Team ID, Apple ID +
  app-specific password — для локального теста и для GitHub Secrets.

## Локальный вспомогательный скрипт

`scripts/release-local.sh` — собирает universal, подписывает, нотаризует,
степлит DMG на твоём Mac (быстрый цикл проверки до CI). Секреты читает из env,
в репозиторий не пишутся.
