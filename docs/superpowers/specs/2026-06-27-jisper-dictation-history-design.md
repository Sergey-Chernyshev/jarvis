# Дизайн: Jisper — отдельная страница истории диктовки

Дата: 2026-06-27
Статус: согласовано (брейншторм + ответы пользователя)

## Цель
Отдельная красивая страница **Jisper** — история **надиктовок (F8)**, с локальным
персистом на диск. История **разговоров** («Hey Jarvis», `source=wake`) — это
ОТДЕЛЬНАЯ сущность, в Jisper НЕ показывается (у диктовки и разговора разные
действия и жизненный цикл). Макет: `docs/superpowers/mockups/jisper.html`.

## Решения (из брейншторма)
- **Размещение:** новая вкладка «Jisper» (⌘3) в ⌘J-панели, рядом с Сессии/Проекты/Статистика.
- **Источник:** только `dictation`. `wake` остаётся в своей ветке (вне Jisper).
- **Хранение:** персист на диск `~/.jarvis[-dev]/dictation-history.json`, хранить
  ВСЁ (safety-cap 5000, чтобы файл не рос бесконечно), load на старте демона,
  save на push/delete/clear.
- **Функции:** поиск по тексту · статистика (сегодня/всего/слов) · группировка по
  дням · → Промпт (LLM enhance, инлайн) · копировать (по одной + всё) · удалить по
  одной · экспорт (.md) · очистить.
- **Дизайн:** дизайн-язык прототипа (тёмное стекло, акцент `#6ca0ff`, `mono` для
  времени), Wispr-строки с ПОЛНЫМ текстом + ховер-действия, нижний action-bar.

## Данные
```rust
struct Transcript { id: u64, text: String, ts: u64, source: String }
```
`id` — стабильный монотонный (для удаления по одной; на load = max(id)+1). Формат
файла: `{ "items": [ {id,text,ts,source}, … ] }`. Новые — первыми (front).

## Бэкенд (`stt/transcripts.rs`, TDD)
- Добавить `id` к `Transcript`; персист `load()/save()` (JSON, дефолтный путь +
  инъекция пути для тестов); `remove(id)`; CAP=5000; save на push/remove/clear.
- Daemon при старте создаёт `Transcripts::load()`.
- IPC: `transcripts_get` (есть, +id), `transcripts_clear` (есть), `transcript_enhance`
  (есть) + новые `transcript_delete{id}`, `transcripts_export → path` (пишет .md в
  `~/.jarvis[-dev]/`, возвращает путь; фронт показывает тост).

## Фронтенд (`ui/renderer.js`, `ui/bridge.js`, `ui/index.html`, `ui/app.css`-блок)
- Новая вкладка/вью `jisper` (⌘3) в табах панели.
- Рендер по макету: search + stats + day-groups + entries (полный текст, время,
  слов) + ховер-действия (→Промпт инлайн, копировать, удалить) + action-bar
  (копировать всё, экспорт, очистить). Только `source==dictation`.
- Команда палитры «Перейти в Jisper».
